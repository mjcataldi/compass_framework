# 03 — Cover Letter Generation

This file governs STRIDE-based cover letters.

## Cover Letter Trigger

Do not generate a cover letter unless the user explicitly requests one.

## Tone

Use a professional, grounded, forward-looking tone. Avoid exaggerated enthusiasm, flattery, or generic filler.

## Source Requirements

A cover letter must be derived from:

- The latest approved source resume or CV
- The target job description or recruiter message
- The current STRIDE analysis, when available
- User-provided motivation or constraints

## Structure

A strong STRIDE cover letter should include:

1. A direct opening that names the role and positions the candidate clearly
2. One or two paragraphs connecting source-backed experience to the role's core needs
3. A concise closing that reinforces fit and forward-looking value

## Avoid

Do not include:

- Unsupported claims
- Overly enthusiastic openings
- Long autobiography
- Repetition of the full resume
- STRIDE analysis notes
- Compensation strategy unless the user specifically asks for negotiation language
- Generic statements that could apply to any candidate

## Recommended Style

Prefer:

- Calm confidence
- Concrete alignment
- Role-specific evidence
- Clear business and technical value
- Concise paragraphs

## Mission-Driven Roles

For mission-driven, public-sector, civic-tech, healthcare, nonprofit, education, or social-impact roles, it is acceptable to include motivation when the source evidence or user's stated preferences support it.

Keep mission language grounded. Do not overdo it.
